# PolancoWatch Backend

## Overview
PolancoWatch backend is a lightweight, high‑performance ASP.NET Core 8 API that collects system metrics from Linux servers, evaluates alert thresholds, and streams real‑time telemetry to the frontend via SignalR.

## Features
- **CPU, Memory, Disk, Network** metrics collected every 2.5 seconds.
- **Process monitoring** with optional host PID namespace (`pid: host`).
- **Alert engine** with customizable thresholds.
- **SignalR** broadcasting for instant UI updates.
- **Docker‑ready** multi‑stage build and Docker Compose support.

## Prerequisites
- Docker & Docker‑Compose (recommended for production).
- .NET 8 SDK (for local development).
- SQLite (bundled, no external DB required).

## Local Development
```bash
# Clone the repo
git clone https://github.com/yourusername/PolancoWatch.git
cd PolancoWatch/PolancoWatch_backend

# Build and run (Docker)
docker build -t polancowatch-backend .
docker run -p 80:80 polancowatch-backend

# Or run directly with .NET
dotnet run --project PolancoWatch.API/PolancoWatch.API.csproj
```
The API will be reachable at `http://localhost` (or the host IP).

## Environment Variables
| Variable | Description | Default |
|----------|-------------|---------|
| `ASPNETCORE_ENVIRONMENT` | ASP.NET environment (Development/Production) | `Production` |
| `ASPNETCORE_URLS` | URLs the Kestrel server listens on | `http://+:80` |
| `VITE_API_URL` | **Not used** in backend – only for frontend build‑time.

## Docker Compose Deployment (recommended)
1. Ensure the `docker-compose.yml` at the repository root contains the backend service with `pid: "host"` and `privileged: true` (already added).
2. In Dokploy, create a **Compose** application and point it to this repository.
3. Deploy – Dokploy will build the image and start the container exposing port 80.

### Deploying the Frontend with Dokploy
1. Add the frontend service to the same Compose file or create a separate Compose app.
2. Set the build‑time argument `VITE_API_URL` to the domain generated by Dokploy (e.g., `http://myapp.traefik.me`).
3. Deploy the frontend; it will automatically connect to the backend via SignalR.

## Architecture Overview
- **API Container** – .NET 8 runtime, runs `SystemMetricsCollector` service, broadcasts via SignalR.
- **Web Container** – Nginx serving the Vite‑built SPA.
- **Database** – SQLite file stored inside the API container (persisted via a volume if needed).

PolancoWatch is a beautiful, lightweight, self-hosted system monitoring platform for Linux servers. It provides real-time telemetry streaming and an alerting system, designed with a stunning Glassmorphism UI.

![PolancoWatch Dashboard](https://via.placeholder.com/1200x800.png?text=PolancoWatch+Dashboard)

## 🚀 Features
- **Real-Time Telemetry:** CPU, Memory, Disk, and Network usage streamed via SignalR.
- **Lightweight Backend:** Built on ASP.NET Core 8 reading directly from the Linux `/proc` filesystem.
- **Modern Dashboard:** React + Vite frontend styled with Tailwind CSS v4 featuring glassmorphism and animations.
- **Alerting Engine:** Built-in threshold evaluation and event notification system out-of-the-box.
- **Easy Deployment:** Fully containerized with Docker and Docker Compose.

## 🏗️ Tech Stack
- **Backend:** C#, .NET 8, ASP.NET Core Web API, EF Core, SQLite, SignalR
- **Frontend:** TypeScript, React, Vite, Tailwind CSS, Recharts, Lucide Icons
- **Deployment:** Docker, Nginx

## 📦 Installation (Docker)

The easiest way to deploy PolancoWatch is via Docker Compose.

1. Clone the repository:
   ```bash
   git clone https://github.com/yourusername/PolancoWatch.git
   cd PolancoWatch
   ```

2. Start the stack:
   ```bash
   docker-compose up -d
   ```

3. Access the Dashboard:
   - Open your browser and navigate to `http://localhost` (or your server IP).
   - **Default Credentials:**
     - Username: `admin`
     - Password: `admin`
   > ⚠️ **Important:** Please change the default admin password immediately after your first login!

## ⚙️ Architecture Overview
- **API Container:** Runs the .NET Background Services that scrape server metrics every 2.5 seconds, evaluate alert thresholds, and broadcast telemetry via SignalR.
- **Web Container:** Serves the highly optimized Vite SPA through an Nginx alpine image.
